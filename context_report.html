RE-ENFORCE App Context Report

This document summarizes the relevant project context for the map application in html_app/index.html. It is based on the RE-ENFORCE project application, the RE-ENFORCE Harmonized Database repository, and the Science article on future forest disturbances in Europe.

1. Project context

RE-ENFORCE is a transnational cooperation project focused on nature-based solutions (NBS) for restoring degraded forests across Central Europe. The project aims to support restoration planning, develop transnational strategies, and provide decision support for forest restoration through shared data and tools.

2. Data source context

The app is connected to the RE-ENFORCE Harmonized Database ecosystem, which contains geospatial datasets grouped into themes such as:

  • LAND COVER (e.g. CORINE land cover)
  • BOUNDARY (administrative and project area shapefiles)
  • CLIMATE (CHELSA climatic data, growing degree days, aridity, etc.)
  • TOPOGRAPHY (elevation, slope, aspect, etc.)
  • FOREST (biomass, tree height, species distribution, seed sources)
  • SOIL (soil types, water holding capacity, fertility)
  • VALIDATION DATA (disturbance and forest validation products)

These datasets are hosted on Zenodo and documented in the GitHub repository JCepl/RE-ENFORCE-Harmonized-Database.

3. Scientific context

The Science article “Climate change will increase forest disturbances in Europe throughout the 21st century” provides an important framing for the app:

  • Forest disturbances are projected to increase under all RCP scenarios.
  • Wildfire is the most sensitive disturbance agent, but bark beetles and wind also contribute significantly.
  • Future disturbance patterns will alter forest demography, increasing young forests and reducing old forests.
  • Disturbance hotspots emerge across Europe and require policy and management planning, especially under climate change.
Key implication: The app should be framed as a forest disturbance and restoration decision support tool, not as a purely hex-based map product.

4. App-purpose recommendation

The map app should support the RE-ENFORCE project by making the following transparent:

  • Which scenario is being shown (RCP 2.6, 4.5, 8.5)
  • What year or time horizon is selected
  • Which dataset or derived indicator is displayed
  • How the result relates to disturbance risk, restoration priority, or forest condition
  • How this connects to the RE-ENFORCE Harmonized Database and the wider project aim

5. Current app state and next action

The current app structure is a static Leaflet viewer with a sidebar of controls. It already includes:

  • RCP scenario selection
  • Year slider
  • Opacity control
  • Overlay toggle
  • Click-to-query information panel

Recommended next steps for this app are:

  • Rename UI labels to reflect disturbance / restoration context instead of generic hex language.
  • Link the overlay to actual RE-ENFORCE dataset products or derived disturbance indicators.
  • Add an in-app information panel that explains the project goal and data source.
  • Preserve the current controls while repositioning them around RE-ENFORCE use cases.
